# 环境变量继承

## 问题

修改系统环境变量后，通过 PowerToys 命令面板启动的 pwsh 拿到的仍是旧环境变量；而通过 Windows Terminal 新建 tab 启动的 pwsh 能拿到最新值。同理，VSCode 即使重启也不刷新环境变量。

## 原因

Windows 子进程**继承父进程的环境变量快照**，而非实时读取系统环境变量。PowerToys 长驻后台，其环境变量停留在启动时的快照——从它启动的子进程自然拿到旧值。VSCode 若由 PowerToys 启动，继承的同样是旧快照。

## 解决

**重启父进程**。退出 PowerToys 再重新打开，此时它拿到最新的系统环境变量，后续启动的子进程也能正确继承。
