> 来源：`buffer/Obsidian/Archive/Applications/bitwarden/Session.md`

## 三层架构

Bitwarden CLI 采用严格的三层架构，确保密码数据在传输和存储过程中的安全性：

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   云端服务器     │     │   本地电脑      │     │   内存中（解密）  │
│  (加密数据)      │ ←── │  (加密缓存)     │ ←── │  (可读取密码)    │
└─────────────────┘     └─────────────────┘     └─────────────────┘
       ↑                      ↑                      ↑
       │                      │                      │
    注册账号               bw login               bw unlock
    存储密码            （下载加密数据）           （主密码解密）
                                               ↓
                                        BW_SESSION
                                        临时访问令牌
```

---

## 命令流程

### 1. bw login — 下载数据

```shell
bw login  # 下载加密数据到本地
```

**作用**：
- 连接 Bitwarden 云端服务器
- 使用账号密码验证身份
- 将加密的密码库数据下载到本地缓存

**注意**：此时数据仍是加密状态，无法读取实际密码内容。

### 2. bw unlock — 解密数据

```shell
bw unlock --raw  # 使用主密码解密，获取 BW_SESSION 临时访问令牌
```

**作用**：
- 输入主密码（Master Password）
- 在内存中解密密码库
- 生成 `BW_SESSION` 临时访问令牌

**--raw 参数**：直接输出令牌字符串，方便写入环境变量或脚本使用。

---

## 安全边界

| 层级 | 数据状态 | 安全风险 | 访问方式 |
|------|----------|----------|----------|
| 云端服务器 | 加密 | 低（密文泄露无害） | 账号密码 |
| 本地电脑 | 加密 | 低（密文缓存） | 文件系统 |
| 内存 | 解密 | 高（明文密码） | BW_SESSION |

**关键原则**：
- 主密码从未存储在任何地方，只在 `bw unlock` 时由用户输入
- `BW_SESSION` 令牌只在当前会话有效，退出后失效
- 即使本地缓存被盗，没有主密码也无法解密
