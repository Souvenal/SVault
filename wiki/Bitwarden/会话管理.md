## BW_SESSION 临时访问令牌

`BW_SESSION` 是 Bitwarden CLI 在 `bw unlock` 成功后生成的临时访问令牌。

### 生成方式

```shell
bw unlock --raw
```

输出示例：
```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 使用方式

**方式一：设置环境变量**

```shell
export BW_SESSION=$(bw unlock --raw)
bw list items  # 无需再次输入主密码
```

**方式二：每次命令传递**

```shell
bw list items --session $(bw unlock --raw)
```

## 会话生命周期

```
bw unlock
    │
    ▼
生成 BW_SESSION ──→ 会话开始
    │
    ├── bw list items ✓
    ├── bw get password <id> ✓
    ├── bw create item ... ✓
    │
    ▼
bw lock / 退出终端 / 超时
    │
    ▼
会话结束 ──→ BW_SESSION 失效
```

## 安全注意事项

### ⚠️ 令牌等同于主密码权限

`BW_SESSION` 令牌可以：
- 读取所有解密后的密码
- 创建、修改、删除密码条目
- 执行所有需要解密权限的操作

### ✅ 最佳实践

1. **不要持久化令牌**
   ```shell
   # 错误：写入配置文件
   echo "BW_SESSION=xxx" >> ~/.bashrc  # ❌
   
   # 正确：仅当前会话使用
   export BW_SESSION=$(bw unlock --raw)  # ✓
   ```

2. **使用后主动锁定**
   ```shell
   bw lock  # 清除内存中的解密数据，BW_SESSION 失效
   ```

3. **自动化脚本建议**
   ```shell
   # 每次脚本运行时解锁，执行完毕后锁定
   BW_SESSION=$(bw unlock --raw)
   # ... 执行操作 ...
   bw lock
   ```
