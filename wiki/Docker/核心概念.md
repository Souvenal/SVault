# Docker 核心概念

Docker 的四个核心实体：Image、Container、Dockerfile、Repository。

---

## Image

Container 的模版，包含应用程序及其依赖项。

- 只读
- 用于创建 Container
- 通过 Dockerfile 构建或从 Repository 拉取

## Container

Image 的实例，运行中的应用程序及其环境。

- 可启动、停止、删除
- 相互隔离
- 每个 Container 是独立的运行实例

## Dockerfile

构建 Docker 镜像的文本文件。

- 包含一系列指令（FROM、RUN、COPY 等）
- 定义 Image 的构建步骤
- 通过 `docker build` 执行

## Repository

存储和分发 Docker 镜像的地方。

- Docker Hub 是官方公共 Repository
- 私有 Registry 用于内部镜像管理
- 支持 tag 和 SHA256 标识版本

---

## Docker 工作流程

```
Dockerfile → docker build → Image → docker run → Container
                         ↑
                    docker pull (从 Repository 拉取)
```

1. 构建镜像：编写 Dockerfile，`docker build`
2. 运行容器：`docker run`
3. 管理容器：`docker ps`、`docker stop`、`docker rm`
