# 记忆功能对比

*六款 AI 编程工具的长期记忆与规则配置体系横向对比。*

---

## 总览

AI 编程工具的"记忆"指跨会话持久化用户偏好、项目规范和工作流指令的能力。核心问题：Agent 如何在每次新对话开始时，自动获得你的个人习惯和项目约定？

当前主流工具普遍采用**分层 Markdown 文件**方案——在不同目录层级放置指令文件，会话启动时自动加载。但文件名、层级结构、自动记忆能力差异显著。

---

## 对比总表

| 维度 | CodeBuddy | Cursor | Trae | Claude Code | Codex | OpenCode |
|:---|:---|:---|:---|:---|:---|:---|
| **类型** | IDE + CLI | IDE | IDE | CLI | CLI | CLI |
| **主指令文件** | `CODEBUDDY.md` | `.cursor/rules/*.mdc` + `AGENTS.md` | GUI 面板 | `CLAUDE.md` | `AGENTS.md` | `AGENTS.md` |
| **用户级文件** | `~/.codebuddy/CODEBUDDY.md` | 应用内 User Rules（无磁盘文件） | GUI 面板 | `~/.claude/CLAUDE.md` | `~/.codex/AGENTS.md` | `~/.config/opencode/AGENTS.md` |
| **项目级文件** | `./CODEBUDDY.md` | `.cursor/rules/` | GUI 面板 | `./CLAUDE.md` | 项目根 `AGENTS.md` | 项目根 `AGENTS.md` |
| **本地私有层** | `./CODEBUDDY.local.md` | ❌ | ❌ | `./CLAUDE.local.md` | ❌ | ❌ |
| **模块化规则** | `.codebuddy/rules/*.md` | `.cursor/rules/*.mdc` | ❌ | `.claude/rules/*.md` | Skills 机制 | `opencode.json` instructions |
| **自动记忆** | ✅ Auto Memory + Typed Memory | ✅ Memories (Beta) | ❌ | ✅ MEMORY.md | ❌ | ❌ |
| **规则条件匹配** | YAML frontmatter (glob) | 始终/智能/特定文件/手动 | ❌ | YAML frontmatter (paths) | `.rules` (Starlark) | YAML frontmatter |
| **Skills** | ✅ | ✅ (SKILL.md) | ❌ | ✅ (.claude/skills/) | ✅ (.agents/skills/) | ✅ |

## 各工具详解

### CodeBuddy（IDE + CLI）

记忆体系最丰富的工具之一，提供五层配置 + 自动记忆。

**文件层级**：

| 层级 | 路径 | 用途 | Git |
|:---|:---|:---|:---|
| 用户记忆 | `~/.codebuddy/CODEBUDDY.md` | 所有项目个人偏好 | 仅本地 |
| 用户规则 | `~/.codebuddy/rules/*.md` | 模块化个人规则 | 仅本地 |
| 项目记忆 | `./CODEBUDDY.md` 或 `.codebuddy/CODEBUDDY.md` | 团队共享指令 | 入库 |
| 项目规则 | `.codebuddy/rules/*.md` | 按主题拆分的项目指令 | 入库 |
| 本地项目 | `./CODEBUDDY.local.md` | 本地项目偏好 | 不入库 |

**Auto Memory**：AI 自主决定保存内容，存储在 `~/.codebuddy/memories/{project-id}/`。默认开启。

**Typed Memory**：结构化记忆类型（`user`、`feedback`、`project`、`reference`），使用 YAML frontmatter 分类。

**模块化规则**：`.codebuddy/rules/*.md` 文件支持 YAML frontmatter 控制 `alwaysApply`、`paths` glob 匹配。

**初始化**：`/init` 命令生成项目 `CODEBUDDY.md`。

---

### Cursor（IDE）

项目级规则功能强大，但用户级配置受限。

**文件层级**：

| 层级 | 路径 | 用途 |
|:---|:---|:---|
| 用户级 | 应用内 Settings → User Rules | 所有项目偏好（无磁盘文件） |
| 项目级 | `.cursor/rules/*.mdc` | 规则文件（MDC 格式） |
| 项目说明 | `AGENTS.md` | 项目整体说明 |

**Rules 四种模式**：
- **Always**：始终加载
- **Agent Requested**：AI 智能判断是否需要
- **Auto Attached**：匹配特定文件 glob 时自动加载
- **Manual**：用户手动 `@RuleName` 引用

**Memories**：Beta 功能，跨会话持久化用户偏好。尚不成熟。

**局限**：用户级规则仅存在 GUI 设置中，无法通过 dotfiles 工具（如 chezmoi）跨机器同步。

---

### Trae（IDE）

记忆功能最弱的工具，完全依赖 GUI 面板操作。

**配置方式**：
- **个人规则**：AI 功能管理 → 规则面板 → 新建个人规则
- **项目规则**：同面板中配置项目规则

**限制**：
- 无文件系统级配置，无法通过 Git 共享
- 无自动记忆功能
- 单条规则建议不超过 10,000 字符，总计 20,000 byte
- 官方推荐英语编写规则

**总结**：Trae 的规则系统仍处于早期阶段，对有跨机器同步需求的用户不够友好。

---

### Claude Code（CLI）

记忆体系最完善、层级最清晰的 CLI 工具。参见 [[Claude-Code/Claude-Code|Claude-Code]]。

**文件层级**（从广到窄）：

| 层级 | 路径 | 用途 |
|:---|:---|:---|
| 托管策略 | `/Library/Application Support/ClaudeCode/CLAUDE.md` | 组织统一管理 |
| 用户级 | `~/.claude/CLAUDE.md` | 个人偏好（跨项目） |
| 项目级 | `./CLAUDE.md` 或 `.claude/CLAUDE.md` | 团队共享 |
| 本地私有 | `./CLAUDE.local.md` | 本地项目偏好（不入库） |

**自动记忆**：`~/.claude/projects/<project_hash>/memory/MEMORY.md`，AI 自主学习并记录。会话启动加载前 200 行，详情按需读取。

**模块化规则**：`.claude/rules/*.md`，支持 `paths` glob 条件匹配。

**初始化**：`/init` 命令自动分析代码库生成 `CLAUDE.md`。

---

### Codex（CLI）

OpenAI 的 CLI 工具，采用 `AGENTS.md` 方案，行为规则可编程。

**文件层级**：

| 层级 | 路径 | 用途 |
|:---|:---|:---|
| 全局 | `~/.codex/AGENTS.md` | 个人偏好（跨项目） |
| 全局覆盖 | `~/.codex/AGENTS.override.md` | 覆盖全局配置 |
| 项目级 | Git 根目录向下每层 `AGENTS.md` | 从全局到局部叠加 |

**特色**：
- **行为规则**：`.rules` 文件（Starlark 语言）可精确控制命令执行权限
- **Skills**：`.agents/skills/` 目录，独立的技能机制
- **禁用**：`codex --no-project-doc` 或 `CODEX_DISABLE_PROJECT_DOC=1`

**局限**：无本地私有配置层，无自动记忆功能。

---

### OpenCode（CLI）

开源 CLI 工具，兼容 Claude Code 配置。参见 [[OpenCode/OpenCode|OpenCode]]。

**文件层级**：

| 层级 | 路径 | 用途 |
|:---|:---|:---|
| 全局 | `~/.config/opencode/AGENTS.md` | 个人偏好（跨项目） |
| 项目级 | 项目根 `AGENTS.md` | 团队共享 |
| Claude 回退 | `~/.claude/CLAUDE.md` | 兼容 Claude Code 配置 |

**特色**：
- **opencode.json instructions**：支持引用本地文件（glob）、远程 URL，实现模块化规则
- **Claude Code 兼容**：当 `AGENTS.md` 不存在时回退读取 `CLAUDE.md`
- **环境变量控制**：`OPENCODE_DISABLE_CLAUDE_CODE=1` 禁用兼容

**局限**：无本地私有层，无自动记忆功能。

---

## 关键差异总结

### 用户级配置（跨项目个人偏好）

这是解决"不自动 commit"等个人习惯的核心层级：

| 工具 | 配置方式 | 可 dotfiles 同步 |
|:---|:---|:---|
| CodeBuddy | `~/.codebuddy/CODEBUDDY.md` + `rules/*.md` | ✅ 文件系统 |
| Cursor | GUI Settings → User Rules | ❌ 仅 GUI |
| Trae | GUI 规则面板 → 个人规则 | ❌ 仅 GUI |
| Claude Code | `~/.claude/CLAUDE.md` | ✅ 文件系统 |
| Codex | `~/.codex/AGENTS.md` | ✅ 文件系统 |
| OpenCode | `~/.config/opencode/AGENTS.md` | ✅ 文件系统 |

**结论**：CLI 工具全部支持文件系统级用户配置，可配合 [[chezmoi/chezmoi|chezmoi]] 等 dotfiles 工具跨机器同步。IDE 工具中只有 CodeBuddy 支持文件系统级用户配置。

### 本地私有层

仅 CodeBuddy 和 Claude Code 提供 `*.local.md` 本地私有配置——不入 Git 但与项目绑定。适合存放本地测试路径、个人调试偏好等。

### 自动记忆

仅 CodeBuddy、Cursor、Claude Code 支持某种形式的自动记忆。其中 CodeBuddy 的 Typed Memory 分类最细致，Claude Code 的按需加载策略最节省上下文。
