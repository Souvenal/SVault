# 用户级配置指南

*如何让 AI Agent 在每次对话中自动注入你的个人习惯，且与项目无关。*

---

## 问题场景

你有一些跨项目的个人工作习惯，例如：
- 不要自动执行 `git commit`，必须我自己 review 后手动提交
- 不要自动执行 `git push`
- 代码注释使用中文
- 提交信息使用 Conventional Commits 格式

这些偏好**与具体项目无关**，你希望无论打开哪个项目，Agent 都能自动遵守。

---

## 各工具配置方法

### CodeBuddy

编辑 `~/.codebuddy/CODEBUDDY.md`：

```markdown
# 个人工作流偏好

## Git 规范
- 绝对不要自动执行 git commit，所有变更必须由我 review 后手动提交
- 不要自动执行 git push
- 修改文件前先确认变更范围

## 代码风格
- 代码注释使用中文
- 提交信息使用 Conventional Commits 格式
```

模块化方式——创建 `~/.codebuddy/rules/git-workflow.md`：

```markdown
---
alwaysApply: true
---
# Git 工作流规则

- 绝对不要自动执行 git commit
- 不要自动执行 git push
- 所有变更必须由我 review 后手动提交
```

### Cursor

1. 打开 Settings → Rules
2. 在 **User Rules** 区域添加规则文本
3. 不支持文件系统级配置，无法通过 dotfiles 工具同步

```
Never execute git commit automatically. All changes must be reviewed by me before committing. Never execute git push.
```

> Cursor 的 User Rules 仅存在于 GUI 中，跨机器需手动重新配置。

### Trae

1. 点击右侧 AI 功能管理设置按钮
2. 进入规则面板 → 新建个人规则
3. 填写规则内容

```
Never execute git commit automatically. All changes must be reviewed by me before committing.
```

> Trae 规则建议用英语编写效果更好，且仅通过 GUI 配置。

### Claude Code

编辑 `~/.claude/CLAUDE.md`：

```markdown
# 个人工作流偏好

- 绝对不要自动执行 git commit，所有变更必须由我 review 后手动提交
- 不要自动执行 git push

## 代码风格
- 代码注释使用中文
- 提交信息使用 Conventional Commits 格式
```

模块化方式——创建 `~/.claude/rules/git-workflow.md`：

```markdown
---
alwaysApply: true
---
# Git 工作流规则

- 绝对不要自动执行 git commit
- 不要自动执行 git push
- 所有变更必须由我 review 后手动提交
```

### Codex

编辑 `~/.codex/AGENTS.md`：

```markdown
# 个人编码偏好

## Git 规范
- 绝对不要自动执行 git commit，所有变更必须由我 review 后手动提交
- 不要自动执行 git push
```

### OpenCode

编辑 `~/.config/opencode/AGENTS.md`：

```markdown
# 个人工作流偏好

## Git 规范
- 绝对不要自动执行 git commit，所有变更必须由我 review 后手动提交
- 不要自动执行 git push
```

或者通过 `opencode.json` 引用外部规则文件：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "instructions": [
    "~/.config/opencode/rules/git-workflow.md"
  ]
}
```

> OpenCode 会自动回退读取 `~/.claude/CLAUDE.md`，如果你已经配置了 Claude Code 的用户级规则，OpenCode 也会继承。

## 跨机器同步策略

对于支持文件系统级配置的工具，可以配合 [[chezmoi/chezmoi|chezmoi]] 管理：

```bash
# 用 chezmoi 管理各工具的用户级配置
chezmoi add ~/.claude/CLAUDE.md
chezmoi add ~/.codex/AGENTS.md
chezmoi add ~/.config/opencode/AGENTS.md
chezmoi add ~/.codebuddy/CODEBUDDY.md
chezmoi add ~/.codebuddy/rules/
```

对于 Cursor 和 Trae，由于用户规则仅在 GUI 中配置，无法通过 dotfiles 工具自动同步，需要在每台机器上手动设置。

---

## 编写有效规则的技巧

1. **简洁具体**：用"绝对不要自动执行 git commit"而非"谨慎使用 git"
2. **控制大小**：建议单文件不超过 200 行，过长会稀释遵从度
3. **结构清晰**：用 Markdown 标题和列表组织
4. **模块化**：按主题拆分规则文件（git-workflow.md、code-style.md、testing.md）
5. **条件匹配**：利用 YAML frontmatter 的 `paths` 字段，让规则仅在特定文件时触发

## 配置验证

配置完成后，建议验证 Agent 是否正确加载了规则：

| 工具 | 验证方式 |
|:---|:---|
| CodeBuddy | `/memory` 命令查看已加载文件 |
| Claude Code | `/memory` 命令查看已加载文件 |
| OpenCode | 检查会话启动日志 |
| Codex | 无直接验证方式 |
| Cursor | 无直接验证方式 |
| Trae | 无直接验证方式 |
