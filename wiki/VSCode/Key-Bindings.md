# VSCode 按键优先级

VSCode 按键处理流程。

---

## 按键处理优先级

```
按键事件 → 全局快捷键系统 → 匹配绑定? → 执行命令 → 终止
                                    ↓ 否
                              当前组件处理
```

1. **全局快捷键系统最先捕获**：所有按键先被顶层快捷键系统处理
2. **匹配绑定规则**：遍历绑定，找「按键匹配 + when 条件 true」
3. **执行并终止**：找到匹配立即执行命令，**停止向下传播**
4. **无匹配才传递**：未匹配才传给聚焦组件（编辑器、插件等）

> 这意味着全局绑定会拦截按键，插件（如 vscode-neovim）无法收到被拦截的按键。
