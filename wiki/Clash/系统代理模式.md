# Clash 系统代理模式

Clash 等现代代理软件的"系统代理"开关，操作的是 Windows 的"使用设置脚本（[[PAC|PAC]]）"选项。

---

## 工作原理

开启系统代理时：

1. Clash 在本地启动 HTTP 服务器（默认 `127.0.0.1:7890`）
2. 打开 Windows 的"使用设置脚本"，填入 `http://127.0.0.1:7890/pac`
3. Clash 根据模式生成不同的 PAC 脚本

关闭系统代理时自动关闭"使用设置脚本"。

## PAC 模式（默认）

- 打开"使用设置脚本"，填入 Clash 的 PAC URL
- Clash 生成包含所有分流规则的 PAC 脚本，实现智能分流

## 全局模式

- **一样**操作"使用设置脚本"，不是切换到手动代理
- 唯一区别：生成的 PAC 脚本让所有流量都走代理：

```javascript
function FindProxyForURL(url, host) {
    return "PROXY 127.0.0.1:%mixed-port%; SOCKS5 127.0.0.1:%mixed-port%; DIRECT;";
}
```

`%mixed-port%` 是 Clash 模板变量，运行时替换为实际端口。

## 什么时候用手动代理？

PAC 和全局模式都操作"使用设置脚本"。只有手动代理需要单独配置：

- 非常老旧的软件、命令行工具不支持 PAC
- 部分 UWP 应用绕过 PAC 但遵守手动代理

详见 [[Windows-Use/代理设置]] 中的对比分析。

## TUN 模式

对于完全不读系统代理、也不支持环境变量的软件，开启 Clash 的 **TUN 模式**，在网络层直接接管流量。
